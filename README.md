## Введение

Данная лабораторная работа направлена на освоение принципов модульного тестирования в Python с применением библиотеки `unittest`. Задание базируется на ранее реализованных функциях для фигур — круга, треугольника, квадрата и прямоугольника. В работе реализованы автоматические unit-тесты и отчёт о результатах тестирования.

---

## Цели и задачи тестирования

**Цель:**  
Проверка корректности работы программных функций, реализующих вычисление площади и периметра (длины окружности) для следующих геометрических фигур: круг, треугольник, квадрат, прямоугольник.

**Задачи:**  
- Обеспечить корректность вычислений при корректных и некорректных входных данных.
- Проверить обработки граничных значений (например, нулевые/отрицательные значения).
- Зафиксировать результаты тестирования в автоматическом режиме.
- Провести анализ ошибок (если будут обнаружены).

---

## Описание тестируемого продукта

Тестируемый проект состоит из следующих модулей:
- **triangle.py** — вычисление площади (по формуле Герона) и периметра треугольника с проверкой корректности сторон.
- **square.py** — функции для квадрата: площадь и периметр.
- **rectangle.py** — функции для прямоугольника: площадь и периметр.
- **circle.py** — функции для круга: площадь и длина окружности (периметр).

Каждая функция включает проверки валидности входных данных.

---

## Область тестирования

В тестировании охвачены функции:
- Площадь и периметр треугольника (`triangle.py`)
- Площадь и периметр квадрата (`square.py`)
- Площадь и периметр прямоугольника (`rectangle.py`)
- Площадь и периметр круга (`circle.py`)

Тестируются как разделы с верными входными параметрами, так и обработка ошибок (например, некорректные или отрицательные значения сторон, невозможные треугольники и прочее).

---

## Стратегия тестирования

- **Тип тестирования:**  
  - Модульное (unit) тестирование.
  - Ручное тестирование на граничных значениях.
- **Среда:**  
  - Python 3.x, модуль `unittest`.
- **Тесты включают:**  
  - Корректные варианты значений (positive tests).
  - Проверку ошибок при неправильных или невозможных параметрах.
  - Граничные условия (нулевые и отрицательные параметры).
- **Запуск тестов:**  
  Все тесты запускаются единым скриптом `run_tests.py`.

## Листинг исходного кода тестов

### Пример: Тесты для треугольника (triangle.py)

```python
import unittest
from triangle import area, perimeter

class TestTriangle(unittest.TestCase):
    def test_area_valid(self):
        self.assertAlmostEqual(area(3, 4, 5), 6.0)
    def test_area_invalid(self):
        with self.assertRaises(ValueError):
            area(1, 2, 10)  # несуществующий треугольник
    # ... другие тесты ...
```

Аналогичные тесты реализованы для квадратов, прямоугольников и кругов.

Общий запуск тестов:
```sh
python run_tests.py
```
